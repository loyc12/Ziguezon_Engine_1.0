======== PHASE 4 - Agent Refactor & Government ========


[ ] 1. BuildQueue rework + agent-owned build decisions    // construction refactor pass B
       - Replace BuildQueue's fixed-array shape with a proper queue data
         structure ( current array-with-shift in removeEntryAmount makes
         large-scale extensions error-prone )
       - Add `requester : Agent` to BuildEntry ( POP, IND, INF, GOV, COM
         valid as requesters ; MNT/BLD/GEN/NAT not )
       - Reservation at order time, with FRACTIONAL support : if the requester
         cannot afford the full bundle, lock what is affordable and let the
         entry accumulate funding over subsequent ticks ; smooths demand
         instead of producing spikes
       - BuildEntry gains : reservedRes, outstandingUnits, effortRemaining
       - BuildQueue caches totalReservedRes for O(1) reads in calcBldMaxDelta
         ( updated incrementally on add / partial-reserve / unit-complete /
         cancel )
       - Parallel processing per tick : GOV-requested entries get capacity
         first, remaining capacity distributed across non-GOV entries
         ( allocation policy itself a future player-facing lever )
       - Financial transaction settled at order-time prices for the locked
         portion ; cancellation refunds locked-but-unbuilt resources
       - tryBuild loses its consumeParts branch ; the queue handles all
         resource deduction
       - Industries decide whether to enqueue / dequeue / liquidate from their
         own SAVINGS, MARGIN, ACT_TRGT, workAccess
       - Refactor the AUTO_BUILD_* constants into per-agent variables
       - Hook for agent-driven order cancellation when SAVINGS / MARGIN
         deteriorate ( frees locked resources back to operations )
       - Infrastructure decisions remain "public" ( covered by gov, see #5 )
       - Replaces the role of debugAutoBuild ( retired in #5 below )

[ ] 2. Industry capital accumulation
       - Industries spend SAVINGS on construction orders ( pay parts at PRICE )

[ ] 3. EconLoc activation hook

       - Honour the per-Pop/Ind/Res rate grids already defined in

[ ] 5. Government subsidies & grants pass

======== PHASE 6 - Dynamics & Polish ========


[ ] 1. Economic expectations
       - Smoothed moving-average of price and demand per ResType
       - Industries plan capacity off the smoothed values to dampen oscillation

[ ] 2. Inflation drivers
       - Drive inflationRate from growth/decay of total monetary mass
       - Apply inflation to inf, pop, AND gov SAVINGS ( currently only
         industry - see economy.zig L838 )
       - Currently a constant 1.0

[ ] 3. Variable tick rate / aggregation
       - Coarse tick when populations cross thresholds
       - Aggregate "background" economies whose state barely changes

[ ] 4. Strategic reserves
       - Government stockpile per ResType, fed by tax-in-kind or SPEND_RES
       - Smooths shocks ; can be released during shortages

[ ] 5. Migration between economies
       - Pop flows along trade routes ; driven by welfare differential
       - Limited by transport capacity ( depends on Phase 5 #4 vessels )
