======== PHASE 4 - Agent Refactor & Government ========


[ ] 1. Agent-owned build/decay decisions
       - Move debugAutoBuild out of Economy
       - Industries decide whether to expand/contract from their own SAVINGS,
         margin, and ACT_TRGT
       - Infrastructure decisions remain "public" ( covered by gov, see #3 )
       - BuildQueue gains a "requester" field ( who pays for the build )
       - Replaces todo Phase 6 #1 ( investment signals from prices )

[ ] 2. Industry capital accumulation
       - Industries spend SAVINGS on construction orders ( pay parts at PRICE )
       - Bankrupt industries auto-decay ( SAVINGS goes very negative -> sell
         off some COUNT for parts at AUTO_DECAY_RES_FACTOR like today )

[ ] 3. EconLoc activation hook
       - Currently inactive locations never become active outside debug paths
       - When player ( or gov, or auto ) builds infra in an EconLoc, flip
         isActive ( see bodyComp.zig L157, L169 )
       - Requires a clear definition of "active enough" ( min infrastructure ?
         min population ? )

[ ] 4. Government taxation pass
       - TAX_POP, TAX_IND on profit; TAX_RES VAT on traded resources;
         TAX_LND on used area
       - Inheritance / liquidation : SAVINGS of dead pops and insolvent
         industries route to Treasury ( econSolver L965, L1065 )
       - Each tax routes into Treasury SAVINGS via NET_DELTA accounting
       - Honour the per-Pop/Ind/Res rate grids already defined in
         governmentData.zig

[ ] 5. Government subsidies & grants pass
       - SUB_POP / SUB_IND offset realized expense
       - GRT_POP / GRT_IND inject SAVINGS to specific agents
       - SPEND_BLD : government places its own BuildQueue entries

[ ] 6. Player-facing policy levers
       - Per-EconLoc or per-economy: tax rates, subsidy rates, allocation
         weight overrides, build priorities
       - Sane defaults so a vanilla economy still runs without policy input



======== PHASE 5 - Inter-Economy Trade ========


[ ] 1. Trade demand/supply matching
       - Each economy publishes ( pricePoint, exportCapacity ) per ResType
         and ( pricePoint, importDemand )
       - Solar-system pass matches surplus -> deficit by price differential,
         minus travel cost from ECON_TRAVEL_TABLE ( already populated by
         transferSolver )

[ ] 2. Cargo in flight
       - Trades produce in-flight parcels with arrival time = travel duration
       - Departure economy : exportCapacity consumed, REVENUE booked
       - Arrival economy   : resource added to stock at arrival tick

[ ] 3. Trade tariffs
       - TAX_COM and policy bits on import per resource

[ ] 4. Vessel-mediated transport
       - Replace abstract trade with actual freighter / mass-driver vessels
       - Vessels are built like industry ( PART cost, WORK to operate )
       - Vessel capacity bounds exportCapacity ; vessel turnaround bounds
         throughput



======== PHASE 6 - Dynamics & Polish ========


[ ] 1. Economic expectations
       - Smoothed moving-average of price and demand per ResType
       - Industries plan capacity off the smoothed values to dampen oscillation

[ ] 2. Inflation drivers
       - Drive inflationRate from growth/decay of total monetary mass
       - Apply inflation to inf, pop, AND gov SAVINGS ( currently only
         industry - see economy.zig L838 )

[ ] 3. Variable tick rate / aggregation
       - Coarse tick when populations cross thresholds
       - Aggregate "background" economies whose state barely changes

[ ] 4. Strategic reserves
       - Government stockpile per ResType, fed by tax-in-kind or SPEND_RES
       - Smooths shocks ; can be released during shortages

[ ] 5. Migration between economies
       - Pop flows along trade routes ; driven by welfare differential
       - Limited by transport capacity ( depends on Phase 5 #4 vessels )
